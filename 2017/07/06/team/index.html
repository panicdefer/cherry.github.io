初创团队建设

Posted on 2017-07-06

初创团队的构建

  • 核心成员
    • CTO：负责产品未来总体发展”方向，擅长营销和销售
    • 架构师：用编码将想法变成现实产品
    • 设计师：让代码产品看上去更有吸引力
  • 研发团队人员两大原则
    • 初创企业招人总原则：喜欢日常挑战性工作、能够适应高节奏、有组织性的纷杂工作氛围、并且个人独立工作与团队协作同样高效
    • 创立团队之初，一定按岗位职责招人
  • 团队的责权利
    • 在有限的时间和条件下，达到产品的预期结果就是责任
    • 有权就有责任的担当，要有吃得苦和受得了委屈的心理素质
    • 又相应的鼓励机制
  • 人才渠道
    • 内部推荐
    • 招聘网
    • 其它
  • 目前需要招的候选人
    • 大数据开发工程师
    • android 和 ios

研发体系和流程

  • 需求管理
    • Conflunces
  • 项目规划
    • 在架构和立项层面共同规划
    • 具体项目只让一个团队负责
  • 代码管理
    • github
    • 提交代码必须codeview
  • 技术环境搭建
    • jenkins travis
    • 统一开发流程（CI，CD）
  • 团队成员状态实时透明
    • trello
    • github project
  • 跟踪迭代进度风险
    • findbugs
  • 全程参与修复产品缺陷
    • JIRA
  • 团队知识的沉淀
    • wiki

初创团队经验

  • 根据团队能力量体裁衣，技术简单化，利用开源和成熟的产品，业务与基础设施适当的平衡和取舍，，系统先做出来，抢占市场入口，人员需要充分沟通，让团队成员理解一直，然后要有很强的执行力。
